Data & privacy

Data ownership

SteadyPlan stores your financial data locally in a SQLite database. There is no cloud sync.

In Docker, the database lives under the mounted /app/data volume (inside the container: /app/data/finance.db). On Unraid, this typically maps to a host folder such as /mnt/user/appdata/steadyplan (or whatever path you mounted).

SteadyPlan does not send account balances, budgets, goals, or history to a cloud service. Optional live price lookup may send ticker symbols/identifiers to external price sources.

JSON export is a per-user safety copy. Whole-instance recovery still depends on backing up the full app data directory.

Backup and restore at a glance
  • Local SQLite database: the live SteadyPlan data store for this instance.
  • JSON export: portable, per-user, and useful before risky actions.
  • Whole-instance SQLite backup: database copy for all users, stored under data/backups.
  • Whole app data directory backup: the real disaster-recovery copy, including finance.db, secret_key.txt, and backups/.
  • Restore validation: checks a JSON file without changing data.
  • Restore commit: replaces this user’s data only after confirmation.
  • Delete user data: removes this user’s finance data, not the login account.

Network exposure

LAN/VPN by default

Safest default is LAN/VPN-only. If you expose SteadyPlan publicly, use HTTPS via a trusted reverse proxy or tunnel, use strong passwords, and consider adding an extra auth layer at the proxy/tunnel.

Only set TRUST_PROXY_HEADERS=1 when SteadyPlan is behind a trusted reverse proxy that sets X-Forwarded-* headers.

This is guidance, not an automated security audit.
