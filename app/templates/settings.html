Data & privacy

Data ownership

SteadyPlan stores your financial data locally in a SQLite database. There is no cloud sync.

In Docker, the live database is /app/data/finance.db inside the mounted /app/data volume. On Unraid, that usually maps to a host path such as /mnt/user/appdata/steadyplan.

SteadyPlan keeps balances, budgets, goals, and history local. Optional live price lookup may still send ticker symbols/identifiers to external price sources.

When you only need a safety copy for one user, start with a JSON export. Whole-instance recovery still depends on backing up the full app data directory.

What each safety copy is for
  • Local SQLite database: the live SteadyPlan data store for this instance.
  • Per-user JSON export: portable safety copy for one user before restore, delete, or larger edits.
  • Whole-instance SQLite backup: copy of finance.db for all users, stored under data/backups.
  • Whole app data directory backup: the real disaster-recovery copy, including finance.db, secret_key.txt, and backups/.
  • Restore check: validates a JSON file without changing data.
  • Restore overwrite: replaces this user’s data only after confirmation.
  • Delete this user’s finance data: removes finance data for this user only, not the login account.

Only set TRUST_PROXY_HEADERS=1 when SteadyPlan is behind a trusted reverse proxy that sets X-Forwarded-* headers.

This is guidance, not an automated security audit.

Trading 212 broker snapshot access (beta)

Save a broker snapshot connection

Start by saving a broker snapshot connection. SteadyPlan only tests read-only broker snapshot access here and stores the key pair encrypted on this server using this app's secret key.

SteadyPlan's own price service and manual/CSV imports stay in place. This connection is for broker snapshot sync groundwork, not order placement or money-moving.

SteadyPlan can keep more than one broker snapshot connection, so separate Invest and Stocks ISA accounts can be saved side by side.

Public API currently supports Invest and Stocks ISA only. Cash ISA and SIPP accounts should stay manual/CSV-tracked for now.
